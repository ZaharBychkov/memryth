# MEMRYTH: логика приложения и ответственность модулей

Этот документ описывает текущую реализацию приложения в репозитории `Memryth_dart`, чтобы README и GitHub-страница отражали фактическое состояние проекта.

## Общая идея

MEMRYTH — это персональная библиотека текстовых записей. Пользователь хранит не только цитаты, но и:

- собственные мысли
- длинные фрагменты текста
- комментарии к сохраненным записям

Приложение работает офлайн и использует локальное хранилище Hive.

## Как устроены данные

### `Tag`

Файл: `lib/models/tag.dart`

Назначение:

- хранит идентификатор тега
- хранит отображаемое имя тега

### `Quote`

Файл: `lib/models/quote.dart`

Несмотря на имя модели, сейчас это универсальная запись приложения. Она хранит:

- `id` — уникальный идентификатор
- `text` — основной текст записи
- `author` — автор или собеседник
- `tagIds` — список связанных тегов
- `typeKey` — тип записи: `quote`, `thought`, `excerpt`
- `createdAt` — пользовательская дата создания
- `updatedAt` — дата последнего изменения
- `isFavorite` — пометка избранного
- `sourceTitle` — название источника
- `sourceDetails` — детали источника
- `note` — личная заметка

Метод `copyWith()` используется при редактировании, переключении избранного и обновлении даты.

## Локальное хранение

### `main.dart`

При запуске приложение:

- инициализирует Flutter
- запускает Hive
- регистрирует адаптеры `TagAdapter` и `QuoteAdapter`
- открывает боксы `tags`, `quotes`, `settings`
- создает контроллер настроек
- передает настройки в `AppSettingsScope`

### Репозитории

Файлы:

- `lib/repositories/quote_repository.dart`
- `lib/repositories/tag_repository.dart`

Назначение:

- читать все записи и теги
- сохранять изменения
- удалять записи
- подписываться на изменения Hive через `watch()`

Это тонкий слой доступа к данным без бизнес-логики.

## Логика списка записей

### `QuoteController`

Файл: `lib/contollers/quote_contoller.dart`

Это главный слой логики для главного экрана. Он отвечает за:

- загрузку записей и тегов из репозиториев
- хранение полного списка записей
- формирование карты тегов по `id`
- поиск
- фильтрацию по тегам
- фильтрацию по типам записей
- фильтр по избранному
- сортировку
- генерацию и поддержку случайного порядка
- пересчет отфильтрованного списка после любого изменения

Поддерживаемые сортировки:

- сначала новые
- недавно измененные
- сначала старые
- случайный порядок

Поиск выполняется по:

- тексту записи
- автору
- названию источника
- деталям источника
- заметке
- именам тегов

### `QuoteListViewModel`

Файл: `lib/viewmodels/quote_list_view_model.dart`

Это тонкая оболочка над `QuoteController`. Отдельный класс нужен, чтобы не смешивать экранную модель списка с базовым контроллером.

## Главный экран

### `QuotesScreen`

Файл: `lib/screens/quotes_screen.dart`

Экран отвечает за пользовательский поток работы со списком:

- подписывается на обновления `quotes` и `tags`
- создает `QuoteListViewModel`
- синхронизирует сортировку с настройками приложения
- отображает поиск и фильтры
- показывает список карточек
- открывает экран деталей
- открывает экран создания или редактирования
- открывает боковую панель настроек
- показывает контекстное меню по долгому нажатию
- удаляет запись с подтверждением
- копирует запись в буфер обмена

На карточке доступны:

- открытие записи
- переключение избранного
- нажатие на тег для фильтрации
- длинное нажатие для меню действий

## Создание и редактирование записи

### `QuoteEditViewModel`

Файл: `lib/viewmodels/quote_edit_view_model.dart`

Отвечает за форму редактирования:

- инициализирует контроллеры полей
- подставляет данные существующей записи
- хранит черновые теги
- определяет тип записи
- хранит дату создания
- отслеживает избранное
- проверяет несохраненные изменения через сигнатуру формы
- создает новые теги при сохранении
- переиспользует уже существующие теги без дублей
- собирает итоговую модель `Quote` и сохраняет ее в Hive

Особенности логики:

- пустой основной текст не сохраняется
- теги сравниваются без учета регистра
- дата создания может задаваться вручную
- при любом сохранении обновляется `updatedAt`

### `QuoteEditScreen`

Файл: `lib/screens/quote_edit_screen.dart`

Экран формы показывает:

- выбор типа записи
- выбор даты создания
- переключатель избранного
- поле текста
- поле автора
- поле источника
- поле деталей источника
- поле личной заметки
- список добавленных тегов
- поле добавления нового тега

Также экран блокирует выход без подтверждения, если есть несохраненные изменения.

## Виджеты интерфейса

### `QuoteCard`

Файл: `lib/widgets/quote_card.dart`

Отвечает за карточку записи в списке:

- отображает тип записи
- показывает избранное
- умеет сворачивать и разворачивать длинный текст

- умеет раскрывать список тегов

Карточка учитывает настройки пользователя:

- размер текста
- межстрочный интервал
- плотность карточек
- количество строк в свернутом виде
- показывать ли заметку
- показывать ли автора и источник
- размер тегов

### `QuoteSearchBar`

Поисковая строка с очисткой ввода.

### `SettingsDrawer`

Файл: `lib/widgets/settings_drawer.dart`

Панель настроек, которая управляет:

- языком
- темой
- размером текста записи
- межстрочным интервалом
- размером UI
- плотностью карточек
- размером тегов
- числом строк в свернутой карточке
- сортировкой по умолчанию
- показом заметки на карточке
- показом автора и источника на карточке

## Настройки приложения

### `AppSettings`

Файл: `lib/settings/app_settings.dart`

Описывает все сохраняемые параметры интерфейса и поведения.

### `AppSettingsController`

Файл: `lib/settings/app_settings_controller.dart`

Читает значения из Hive и обновляет их при изменении пользователем.

### `AppSettingsScope`

Файл: `lib/settings/app_settings_scope.dart`

Передает контроллер настроек по дереву виджетов через `InheritedNotifier`.

### `AppStrings`

Файл: `lib/settings/app_strings.dart`

Содержит локализованные строки интерфейса для русского и английского языков.

## Итог

Текущая версия MEMRYTH — это не просто список цитат, а офлайн-приложение для личной текстовой базы знаний. Основные сценарии сейчас такие:

- быстро сохранить мысль, цитату или фрагмент
- снабдить запись контекстом и своей интерпретацией
- позже найти нужный текст по поиску, тегам, типу или избранному
- подстроить интерфейс под собственный способ чтения
