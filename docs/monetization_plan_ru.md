# MEMRYTH: коммерческий и продуктовый план

Дата фиксации: 2026-03-23

Обновлено: 2026-05-07

Статус: документ сохранен как отдельный коммерческий план. Актуальная
техническая дорожная карта находится в `docs/product_roadmap_ru.md` и
`docs/implementation_plan_ru.md`.

Важное уточнение после обсуждения продукта:

- базовый export/import и backup/restore остаются бесплатными, потому что это
  функции доверия к данным;
- PIN/biometric lock не должен становиться Pro-функцией, но для beta он скрыт
  из UI до понятной recovery-модели;
- one-time Pro не должен делать пользователя зависимым от покупки для
  базового сохранения и восстановления библиотеки;
- подписку не запускать, пока нет sync/cloud/AI/web clipper или другого
  серверного слоя с постоянными расходами.

## 1. Что за продукт

MEMRYTH нужно позиционировать не как "еще одно приложение с цитатами", а как:

`private offline commonplace book`

или по-русски:

`личная офлайн-библиотека мыслей, цитат и фрагментов`

Ключевая идея:

- не общий блокнот
- не second brain
- не SaaS для команд
- не AI-продукт
- не менеджер статей

А именно:

- приватное личное пространство
- быстрое сохранение текста и контекста
- удобное возвращение к сохраненному
- локальное хранение без обязательного аккаунта

## 2. Рекомендуемая модель монетизации

### Основная рекомендация

На текущем этапе лучший путь:

`free + one-time Pro unlock` внутри Google Play

Почему именно так:

- приложение пока локальное и не дает recurring value уровня sync/cloud/AI/server
- подписка для локального продукта будет восприниматься как несправедливая
- free-ядро снижает порог входа и помогает собрать установку, отзывы и раннюю аудиторию
- разовый Pro unlock лучше соответствует характеру продукта

### Чего не делать сейчас

Не запускать сразу:

- платную установку как единственный вход
- ежемесячную подписку на локальные функции
- сложную тарифную сетку

### Базовая схема

Бесплатно:

- создание и редактирование записей
- теги
- избранное
- базовый поиск
- базовые фильтры
- светлая и темная тема
- базовые настройки чтения
- export/import
- backup/restore
- PIN/biometric lock после recovery-модели

Pro за разовую покупку:

- Review Mode
- Smart Resurfacing
- Collections / Projects
- Beautiful Export / Share Cards
- Backup History
- Premium Reading / Focus Modes

Больше не считать Pro-функциями:

- export selected
- обычные batch actions
- quick add shortcut
- базовый quick-add widget
- базовый export/import

## 4. Главные слабые места и как их закрывать

### 4.1. Backup/export/import

Статус 2026-05-05: базовый backup/export/import реализован и остается
бесплатным.

Это главный риск для доверия, поэтому функция не должна быть Pro-only.

Если пользователь платит за продукт и потом теряет телефон или переустанавливает систему, он будет считать, что потерял самое ценное: личную библиотеку.

#### Что уже сделано

Минимальный обязательный набор:

- export всех данных в `.json`
- import из `.json`
- ручной backup/restore через файл
- экран "Данные" в настройках
- безопасный merge-режим без перезаписи текущей библиотеки

Не добавлять в beta:

- режим `Заменить текущую библиотеку`;
- полный wipe библиотеки.

#### Как это подавать

Не писать "есть облачная синхронизация", пока ее нет.

Честная формулировка:

- Local backup
- Export your library
- Restore from file

#### Приоритет

Закрыто как критический релизный блок. Ручной smoke test на телефоне выполнен,
находки перенесены в `docs/beta_smoke_findings_ru.md`.

### 4.2. Слишком высокий friction capture

Сейчас запись можно создать только внутри приложения. Для этой категории это слабое место.

#### Что сделать

Первая волна:

- Android share target: сохранить текст из браузера, Telegram, YouTube, книг, заметок
- quick add entry
- отдельная быстрая форма без лишних полей
- автоподстановка текста из внешнего share intent

Вторая волна:

- обработка выделенного текста
- deep link/open as save target
- widget для быстрого добавления

#### Что давать пользователю

Обещание должно звучать так:

- "Сохраняйте важные фрагменты за несколько секунд"

Не так:

- "Импортируем все отовсюду", если такой инфраструктуры еще нет

#### Приоритет

Очень высокий. После backup/export это следующий по важности блок.

### 4.3. Релизная упаковка не готова

Перед релизом надо убрать все, что сигнализирует "dev build".

#### Что исправить

- реальный `applicationId`
- реальный `namespace`
- release signing
- итоговое имя приложения
- финальные иконки
- store screenshots
- короткое и длинное описание для Google Play
- privacy policy page
- contact email

#### Отдельно проверить

- versioning
- package name consistency
- minSdk/targetSdk
- release build без debug signing

#### Приоритет

Обязательный релизный блок.

### 4.4. Первый запуск с demo seed

Для приватного продукта автозаполнение чужими примерами ощущается как мусор.

#### Что сделать

Рекомендуемая замена:

- убрать автосид из прод-версии
- сделать onboarding
- на onboarding дать выбор:
  - начать с пустой библиотеки
  - загрузить demo examples

#### Лучший подход

По умолчанию:

- пустое состояние
- короткое объяснение 3 типов записей
- одна кнопка "Создать первую запись"

#### Приоритет

Высокий. Это влияет на первое впечатление и доверие.

### 4.5. Слабая тестовая база

Сейчас нужен не просто `flutter test passes`, а защита от продуктовых регрессий.

#### Какие тесты добавить в первую очередь

Unit tests:

- сохранение quote/thought/excerpt
- создание тегов без дублей
- фильтрация по типам
- фильтрация по тегам
- поиск по тексту/автору/source/note/tag
- сортировки
- переключение favorites
- сохранение createdAt/updatedAt

Widget tests:

- создание записи
- редактирование записи
- предупреждение о несохраненных изменениях
- пустое состояние

#### Приоритет

Высокий. Это влияет на надежность релиза и скорость дальнейшей разработки.

## 5. Ответы на потенциальные возражения пользователей

Ниже зафиксированы типовые вопросы и решения.

### "Почему нет синхронизации между устройствами?"

#### Краткий ответ сейчас

Потому что MEMRYTH пока local-first продукт.

#### Что делать продуктово

До релиза:

- честно не обещать sync
- дать export/import
- показать, что библиотека не заперта в приложении

После появления аудитории:

- оценить спрос на sync
- если retention хороший, строить optional Sync tier

#### Как формулировать в магазине

- "Offline-first"
- "No account required"
- "Manual backup and restore"

### "Почему нельзя быстро сохранить цитату из браузера/книги/Telegram/YouTube?"

#### Решение

Добавить:

- share intent
- quick add flow
- prefilled create screen

#### Что это даст

- выше ежедневную полезность
- лучшее удержание
- более сильную причину платить за Pro позже

### "Почему нет экспорта в файл?"

#### Решение

Это нужно сделать до монетизации.

Минимальный вариант:

- export/import `.json`

Хороший вариант:

- export selection
- full library export
- backup timestamp

### "Почему после покупки это только Android?"

#### Решение

Сейчас нужно говорить честно:

- Android-first

Не обещать сроки на iOS/Web, пока нет дорожной карты и ресурсов.

Формулировка:

- "MEMRYTH starts on Android. Cross-platform expansion depends on demand."

### "Почему первый запуск заполнен чужими данными?"

#### Решение

Убрать автосид и заменить его:

- пустым состоянием
- onboarding
- необязательной кнопкой "Try demo examples"

### "Почему нет PIN/biometric lock, если тут мои личные заметки?"

#### Решение

Это не Pro-функция. Это часть доверия к личным данным.

Решение после beta:

- вернуть app lock только после recovery-модели;
- не продавать PIN/biometric отдельно;
- честно написать, что это не шифрование базы;
- использовать более медленный KDF вместо простого SHA-256.

### "Почему нет batch-actions: массовые теги, экспорт подборки, объединение тегов?"

#### Решение

Обычные batch-actions уже стали free-core и не должны возвращаться в Pro.

Бесплатно оставить:

- multi-select;
- batch tag assign;
- favorite/unfavorite selected;
- export selected;
- осторожный delete selected с подтверждением.

Pro может расширять это только через более высокоуровневые сценарии:

- Collections / Projects;
- Collection Export;
- Review Mode по коллекции;
- Smart Resurfacing по теме или коллекции.

### "Почему я должен платить повторно, если это локальный продукт без облака?"

#### Решение

Не заставлять платить повторно.

Поэтому сейчас не подписка, а:

- one-time Pro unlock

Подписка допустима только потом, если появятся:

- sync
- cloud backup
- AI
- web clipper service
- серверные функции

## 11. Итоговая зафиксированная стратегия

MEMRYTH запускается как:

- Android-first
- offline-first
- no account required
- free core
- one-time Pro unlock

MEMRYTH не позиционируется как:

- очередной quotes app
- second brain
- AI knowledge platform
- subscription note app без облачной ценности

Главный продуктовый фокус:

- доверие к данным
- скорость сохранения
- удобство возвращения к сохраненному
- ощущение личного приватного пространства
