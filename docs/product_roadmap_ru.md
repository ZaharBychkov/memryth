# MEMRYTH: общий продуктовый roadmap

Дата фиксации: 2026-05-04

Обновлено: 2026-05-05

Цель файла: сохранить общий план развития приложения до покупки Google Play
Developer account, после покупки и после первого публичного запуска.

Этот документ шире, чем `ux_next_steps_ru.md`. UX-файл описывает ближайшие
правки интерфейса. Здесь зафиксированы продуктовая логика, beta/full этапы,
монетизация, Pro, Google Play, будущий sync/cloud/AI-слой и общие технические
долги.

## 1. Позиционирование продукта

MEMRYTH — это не "приложение с цитатами" и не общий блокнот.

Позиционирование:

`личная офлайн-библиотека мыслей, цитат и фрагментов`

Ключевая ценность:

- быстро сохранить важную мысль, цитату или фрагмент;
- добавить контекст: автор, источник, заметка, темы;
- позже спокойно вернуться к сохраненному;
- хранить данные локально без обязательного аккаунта;
- иметь понятный backup/export, чтобы не бояться потери данных.

Главный эмоциональный сценарий:

`Я сохраняю важные мысли в одном месте и возвращаюсь к ним позже.`

## 2. Текущее состояние приложения

Уже реализовано:

- локальное хранение через Hive;
- создание, редактирование и удаление записей;
- типы записей: цитата, мысль, фрагмент;
- автор, источник, заметка, темы, избранное;
- поиск по тексту, автору, источнику, заметке и темам;
- поиск `#тема` как явный поиск по темам;
- фильтры по типу, теме и избранному;
- сортировки: новые, недавно измененные, старые, случайный порядок;
- индекс тем со счетчиками и деревом через `/`;
- русская и английская локализация;
- светлая и темная тема;
- настройки чтения;
- onboarding;
- пустой приватный старт без автоматических demo-записей;
- экспорт всей библиотеки в JSON;
- импорт JSON в безопасном merge-режиме;
- export selected через тот же совместимый JSON-формат;
- PIN lock;
- biometric unlock как опция после PIN;
- privacy policy внутри приложения;
- share-to-app;
- Android quick add shortcut;
- Android home widget для quick add;
- массовый режим:
  - export selected;
  - добавить темы выбранным;
  - убрать темы у выбранных;
  - добавить/убрать избранное;
- обзорный экран MEMRYTH Pro;
- каркас Google Play Billing через `in_app_purchase`;
- локальный Pro entitlement;
- Android package id / namespace: `app.memryth.android`;
- release signing config с fallback на debug key;
- release APK/AAB сборки проходят.

Удалено из beta:

- `Сохранить текущий вид` / saved filters.
- лишнее отдельное поле источника: удалено из UI, модели `Quote`,
  Hive adapter, JSON export/import, demo seed, тестов и актуальных docs.

Причина удаления:

- пользователь не понимал функцию;
- она путала фильтрацию с сохранением записей;
- для beta это лишняя сложность;
- функция вызывала runtime crash в диалоге.
- отдельное поле источника дублировало `Источник` и путало форму записи.

## 3. Что делаем до покупки Google Play Developer account

Пока аккаунт Google Play не куплен, фокус не на биллинге, а на доведении
самого приложения до понятного и надежного состояния.

Закрыто в UX-проходе 2026-05-05:

- пустой старт показывает первое действие, а не инструменты пустой библиотеки;
- доступны `Создать первую запись` и `Импортировать backup`;
- фильтры типов стали `Все / Цитаты / Мысли / Фрагменты / ★`;
- форма новой записи использует `Текст цитаты`, `Текст мысли`,
  `Текст фрагмента` и блок `Контекст`;
- поле `Источник` получило более широкий смысл;
- лишнее отдельное поле источника удалено полностью;
- `Аккаунт / Синхронизация` убраны из настроек до реальной реализации.

Актуальные приоритеты до покупки Google Play:

1. Техническая чистка:
   - `QuoteSortMode` вынесен в `lib/models/quote_sort_mode.dart`;
   - пустой `QuoteListViewModel` удален, `QuotesScreen` использует `QuoteController` напрямую;
   - `lib/controllers` уже используется вместо старого пути;
   - разобраться с untracked tooling-файлами:
     `.agents/`, `.claude/`, `skills-lock.json`;
   - держать `docs/monetization_plan_ru.md` как отдельный коммерческий план,
     но не позволять ему противоречить этому roadmap.

2. Ручное тестирование на телефоне:
   - первый запуск;
   - создание записи каждого типа;
   - редактирование;
   - темы;
   - массовые действия;
   - экспорт/импорт;
   - share-to-app;
   - quick add;
   - PIN/biometric;
   - темная/светлая тема.

3. Store preparation без покупки аккаунта:
   - актуализировать screenshots;
   - подготовить feature graphic;
   - проверить тексты Google Play listing;
   - проверить privacy/data safety формулировки.

## 4. Beta-версия

Идея beta:

- не покупать рекламу сразу;
- не начинать с подписки;
- дать нескольким людям пользоваться приложением;
- собрать реальные непонимания, баги, UX-провалы;
- исправить приложение до публичной загрузки.

Что должно быть в beta:

- локальная библиотека без аккаунта;
- создание/редактирование/удаление записей;
- темы;
- избранное;
- поиск и фильтры;
- настройки чтения;
- export/import;
- PIN;
- массовые действия;
- пустой первый старт;
- понятная форма новой записи;
- без заглушек вроде sync/account.

Что не обязательно для beta:

- Google Play Billing;
- реальные покупки;
- реклама;
- cloud sync;
- AI;
- web clipper;
- paid subscription.

## 5. Google Play: что понадобится

Разовая стоимость:

- Google Play Developer account: около `$25` один раз.

Если приложение остается локальным:

- хостинг приложения в Google Play: `$0/месяц`;
- сервер: `$0/месяц`;
- Firebase/Google Cloud: `$0/месяц`, если нет sync/cloud.

Перед загрузкой в Google Play нужно:

- купить Google Play Developer account;
- создать приложение в Play Console;
- подготовить listing:
  - название;
  - короткое описание;
  - полное описание;
  - privacy policy;
  - screenshots;
  - feature graphic;
- создать managed product `memryth_pro`;
- проверить purchase/restore на internal testing track;
- собрать release AAB;
- пройти content rating / data safety.

## 6. Монетизация

Текущая рекомендованная модель:

`Free + one-time Pro unlock`

Почему не подписка сейчас:

- приложение пока локальное;
- локальные функции не создают ежемесячной серверной ценности;
- подписка на локальный блокнот/библиотеку может восприниматься как
  несправедливая;
- one-time Pro честнее и проще для первого продукта.

Почему не платная установка:

- free-ядро снижает порог входа;
- проще получить первых пользователей;
- проще собрать отзывы;
- меньше недоверия к новому приложению без репутации.

Ориентир по цене:

- стартовый Pro: `$3.99`;
- после отзывов и улучшений можно поднять до `$5.99`.

Это не финальное решение, но хороший первый ориентир.

## 7. Что должно быть бесплатным

Бесплатно должно остаться то, что формирует доверие к продукту:

- создание записей;
- редактирование;
- удаление отдельных записей;
- темы;
- избранное;
- базовый поиск;
- базовые фильтры;
- светлая/темная тема;
- базовые настройки чтения;
- export всей библиотеки;
- import/restore через JSON в merge-режиме;
- PIN/биометрия.

Причина:

- нельзя запирать базовый backup за оплату;
- пользователь должен доверять приложению свои данные;
- приватность и безопасность — часть базового обещания продукта.

## 8. One-time Pro: возможный состав

Pro не должен делать пользователя зависимым от покупки для базового сохранения
мыслей.

Pro должен ощущаться как расширение для активного пользователя.

Кандидаты в Pro:

- export selected;
- batch export;
- массовые действия;
- Android widgets;
- random favorite widget / resurfacing widget;
- backup reminders;
- saved collections;
- resurfacing старых записей;
- расширенные способы организации библиотеки;
- дополнительные premium-настройки интерфейса, если они реально полезны.

Уже реализованные или частично реализованные Pro-кандидаты:

- export selected;
- массовые действия;
- Android quick add widget;
- каркас Pro purchase / restore.

Важно:

- пока нет реального Play Console product id и теста покупки, Pro нельзя
  считать готовым к продаже;
- экран Pro может существовать как каркас, но публично продавать можно только
  после проверки managed product.

## 9. Что не делать в локальном Pro

Не добавлять в первый Pro:

- подписку;
- sync/cloud;
- AI;
- web clipper;
- batch delete;
- полный wipe библиотеки;
- импорт в режиме "заменить все".

Причина:

- sync/cloud/AI требуют отдельной инфраструктуры и могут оправдать подписку,
  но сейчас это другой слой продукта;
- batch delete и wipe опасны для данных;
- replace import может привести к потере библиотеки.

## 10. Будущий subscription tier

Подписку можно рассматривать только позже, если появятся функции с постоянными
расходами или серверной ценностью:

- sync между устройствами;
- cloud backup;
- AI-бот по личной библиотеке;
- web clipper / browser extension;
- серверная обработка или хранение;
- регулярные внешние API.

Правило:

- one-time Pro остается локальным Pro;
- subscription tier должен быть отдельным;
- пользователь, купивший one-time Pro, не должен потерять локальные Pro-функции
  из-за появления подписки.

## 11. Будущие крупные функции

Слой после стабильного локального приложения:

1. Sync/cloud:
   - аккаунт;
   - несколько устройств;
   - cloud backup;
   - конфликт-резолвинг;
   - серверные расходы.

2. AI:
   - поиск/ответы по личной библиотеке;
   - тематическое резюме;
   - помощь в связывании мыслей;
   - только после решения privacy и стоимости.

3. Web clipper:
   - сохранение фрагментов из браузера;
   - расширение;
   - отдельный серверный или локальный сценарий.

4. Resurfacing:
   - случайная старая запись;
   - повторное всплытие избранного;
   - widget на экран телефона;
   - мягкие напоминания без шума.

5. Saved collections:
   - явные подборки записей;
   - не путать с удаленными saved filters;
   - добавлять только если появится понятный сценарий.

## 12. Что считать готовностью к первому публичному релизу

Технически:

- `flutter analyze` чистый;
- `flutter test` зеленый;
- `flutter build apk --release` проходит;
- `flutter build appbundle` проходит;
- release APK вручную проверен на телефоне;
- импорт/экспорт вручную проверен;
- Android share-to-app проверен;
- quick add widget проверен;
- PIN/biometric проверены.

Продуктово:

- первый экран понятен;
- создание записи понятно без объяснения;
- фильтры не выглядят как кнопки создания;
- нет заглушек в пользовательском пути;
- нет лишних отдельных полей для источника;
- базовый backup доступен бесплатно;
- Pro не обещает то, что еще не работает.

Google Play:

- listing готов;
- screenshots готовы;
- privacy policy готова;
- data safety заполнен честно;
- managed product создан и проверен;
- internal testing пройден.
