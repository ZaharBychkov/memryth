# MEMRYTH: практический план реализации

Дата фиксации: 2026-05-01

## 1. К чему стремимся

MEMRYTH должен стать не просто списком цитат, а приватной офлайн-библиотекой личных текстовых фрагментов:

- пользователь быстро сохраняет цитаты, мысли и фрагменты;
- у каждой записи есть контекст: автор, источник, детали источника, заметка, теги;
- сохраненное легко найти через поиск, фильтры, избранное и сортировки;
- данные хранятся локально, без обязательного аккаунта;
- пользователь понимает, как сохранить резервную копию и восстановить библиотеку.

Главный критерий продаваемого MVP: пользователь доверяет приложению свои данные и не боится их потерять.

## 2. Что уже есть

- Создание, редактирование и удаление записей.
- Типы записей: цитата, мысль, фрагмент.
- Автор, источник, детали источника, заметка.
- Теги и избранное.
- Поиск по тексту, автору, источнику, деталям источника, заметке и тегам.
- Фильтры по типам, тегам и избранному.
- Сортировки: новые, недавно измененные, старые, случайный порядок.
- Русский и английский интерфейс.
- Светлая и темная тема.
- Настройки чтения: размер текста, межстрочный интервал, размер интерфейса, плотность карточек, размер тегов.
- Локальное хранение через Hive.

## 3. Чего не хватает до честного релиза

Критично:

1. Экспорт, импорт, backup и restore.
2. Понятный экран "Данные" в настройках.
3. Пустое состояние вместо автоматического demo seed в прод-флоу.
4. Базовый onboarding.
5. Нормальная Android release-конфигурация: package id, namespace, signing.
6. Тесты на основную бизнес-логику.

Важно после критичного блока:

1. Share-to-app и quick add.
2. PIN/biometric lock.
3. Pro unlock.
4. Widgets.
5. Batch actions.
6. Saved collections и resurfacing.

## 4. Ближайший порядок работ

### Этап 1. Доверие к данным

Сделать первым:

- экспорт всей библиотеки в `.json`;
- импорт библиотеки из `.json`;
- ручной backup/restore через файл;
- отдельный блок "Данные" в настройках;
- предупреждение перед импортом;
- формат файла с версией схемы.

Результат: библиотека пользователя не заперта внутри приложения.

### Этап 2. Первый запуск

Сделать:

- убрать автоматическое заполнение demo-записями из прод-старта;
- сделать пустое состояние;
- добавить кнопку "Создать первую запись";
- позже добавить необязательную кнопку "Загрузить примеры".

Результат: приложение ощущается приватным, а не заполненным чужими данными.

### Этап 3. Настройки и удобство чтения

Сделать:

- переработать настройки в понятные секции;
- добавить сброс настроек;
- заменить грубые варианты размера текста на сенсорную шкалу с ограниченными шагами;
- добавить живой preview текста внутри настроек.

Результат: настройки выглядят аккуратнее и дают пользователю контроль без риска сломать интерфейс.

### Этап 4. Релизная подготовка

Сделать:

- заменить `com.example.memryth_dart_project` на реальный package id;
- заменить Android namespace;
- настроить release signing;
- проверить release build;
- подготовить privacy policy, screenshots, feature graphic и описание Google Play.

Результат: приложение технически готово к публикации.

## 5. Как реализовать экспорт

### Где разместить

Экспорт должен находиться в настройках, но не рядом с размером текста как еще одна мелкая визуальная настройка.

Лучше разделить настройки на секции:

1. `Данные`
   - Экспорт библиотеки
   - Импорт из файла
   - Восстановить backup
   - Дата последнего экспорта
2. `Чтение`
   - Размер текста записи
   - Межстрочный интервал
   - Плотность карточек
3. `Интерфейс`
   - Тема
   - Язык
   - Размер интерфейса
   - Размер тегов
4. `Безопасность`
   - PIN/biometric lock, когда появится
5. `Сброс`
   - Сбросить настройки интерфейса

Так пользователь быстро понимает, что backup/export относится к сохранности данных, а не к внешнему виду.

### Формат файла

Минимальный формат:

```json
{
  "app": "MEMRYTH",
  "schemaVersion": 1,
  "exportedAt": "2026-05-01T12:00:00.000Z",
  "tags": [],
  "quotes": []
}
```

В `tags` сохранять:

- `id`
- `name`

В `quotes` сохранять:

- `id`
- `text`
- `author`
- `tagIds`
- `typeKey`
- `createdAt`
- `updatedAt`
- `isFavorite`
- `sourceTitle`
- `sourceDetails`
- `note`

Важно сохранить `schemaVersion`, чтобы будущие версии приложения могли импортировать старые backup-файлы.

### Логика экспорта

Сервис:

- `ExportImportService`
- читает все теги и записи через репозитории;
- собирает JSON;
- сохраняет файл `memryth-backup-YYYY-MM-DD-HH-mm.json`;
- предлагает пользователю сохранить/поделиться файлом через системный Android sheet.

Для реализации на Flutter лучше использовать:

- `path_provider` для временного файла;
- `share_plus` для отправки/сохранения экспортированного файла;
- `file_picker` для выбора файла при импорте.

На Android лучше не строить экспорт вокруг прямого доступа к общей файловой системе и storage permissions. Надежнее использовать системный выбор/шаринг файла.

### Логика импорта

Первый вариант:

- пользователь выбирает `.json`;
- приложение проверяет `app`, `schemaVersion`, наличие `tags` и `quotes`;
- показывает экран подтверждения;
- предлагает режим:
  - `Объединить с текущей библиотекой`;
  - `Заменить текущую библиотеку`.

Для первого релиза можно начать с одного безопасного режима:

- `Объединить`.

Замену всей библиотеки лучше добавить после тестов, потому что это риск потери данных.

### Проверки перед импортом

Нужно проверять:

- файл действительно JSON;
- `schemaVersion` поддерживается;
- записи имеют `id` и `text`;
- даты корректно парсятся;
- теги без имени не импортируются;
- дубликаты тегов по имени не создаются повторно;
- конфликтующие `id` не затирают существующие записи без явного решения.

## 6. Что делать с размером текста

Идея заменить "маленький / средний / большой" на сенсорную шкалу правильная, потому что сейчас это выглядит грубо и дешево.

Но не стоит делать полностью свободный slider от 10 до 40. Пользователь может выставить некрасивое значение и сломать читаемость.

Лучший вариант:

- slider с 5 фиксированными делениями;
- подписи не как основной UI, а как короткое текущее значение;
- живой preview карточки;
- значения примерно: `18`, `20`, `22`, `24`, `26`;
- дефолт: `22`.

Для межстрочного интервала:

- тоже slider с 4-5 делениями;
- значения примерно: `1.25`, `1.35`, `1.45`, `1.55`;
- дефолт: `1.4`.

Для размера интерфейса:

- лучше оставить 3 режима или сделать осторожный slider с 3 делениями;
- интерфейсный текст сильнее влияет на переполнение кнопок и фильтров, поэтому его нельзя делать слишком свободным.

## 7. Сброс настроек

Сброс нужен. Он должен быть внизу настроек, отдельно от обычных параметров.

Минимально:

- "Сбросить настройки интерфейса";
- подтверждение через dialog;
- сбрасываются тема, язык, размер текста, межстрочный интервал, плотность карточек, размер тегов, preview-параметры и сортировка по умолчанию;
- пользовательские записи и теги не трогаются.

Текст должен ясно говорить:

`Записи, теги и заметки не будут удалены. Сбросятся только настройки внешнего вида и чтения.`

Удаление всей библиотеки не добавлять в первый релиз. Если когда-нибудь понадобится, оно должно быть отдельно, с очень жестким подтверждением.

## 8. Что делать бесплатным, а что Pro

Бесплатно:

- базовый export всей библиотеки;
- базовый import/restore;
- сброс настроек;
- настройки чтения;
- создание и поиск записей.

Pro:

- export selected;
- batch export;
- автоматические backup reminders;
- PIN/biometric lock;
- widgets;
- batch actions;
- saved collections;
- resurfacing.

Причина: базовый export нельзя запирать за оплату. Для приватного локального продукта это вопрос доверия, а не premium-роскошь.

## 9. Следующий практический шаг

Начинать стоит с блока "Данные" в настройках:

1. Добавить секцию `Данные` в полноценный экран настроек.
2. Добавить сервис export/import.
3. Добавить зависимости `path_provider`, `share_plus`, `file_picker`.
4. Реализовать export `.json`.
5. Реализовать import `.json` в режиме merge.
6. Добавить тесты на JSON-сериализацию и импорт без дублей.

После этого можно переходить к переработке настроек чтения на sliders и reset settings.
